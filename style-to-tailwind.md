# style から Tailwind へ置き換えられる箇所の棚卸し

## 対象範囲

- `src` / `public` 配下を対象に、`style=...` と `<style>...</style>` を検索
- `node_modules`、build 出力、レポート類は除外
- 検出件数:
  - `style=`: 82 箇所
  - `<style>` ブロック: 2 箇所

## 判定基準

- `可`: `style` を消して Tailwind クラスへ直接置換できる
- `一部可`: static な部分は Tailwind 化できるが、動的値やコンポーネント API の都合で `style` を完全には消せない
- `不可寄り`: 任意の `style` 受け渡しや動的値注入が主目的で、Tailwind 置換は不自然

## まとめ

- そのまま素直に Tailwind 化しやすいのは:
  - `src/pages/dev/thumb_generator/product/[characterId].astro`
  - `src/pages/how_to_use.md`
  - `src/pages/product/_TopContainer.astro` の `::before`
  - `src/pages/dormitory/top_illust/[topIllustIndex].astro`
  - `src/pages/song/_CharacterCard.tsx` の `color: inherit`
  - `src/pages/dormitory/call_names/index.astro` の対角線背景
- `style` が残りやすいのは:
  - 動的な `backgroundImage`
  - キャラクターごとに変わる `color` / `borderColor` / `backgroundColor`
  - 既存 Tailwind class に値を渡すための CSS 変数注入
  - コンポーネントの任意 `style` 転送
  - `Button` のように CSS 変数注入を前提にした API
- 完全になくすには、単純な class 置換だけでなく:
  - CSS 変数を `data-*` やラッパーへ寄せる
  - コンポーネント API を `class` / `tone` 寄りに寄せる
  - 動的背景画像用の受け皿を用意する
    といった設計変更が必要な箇所もあります。
